## カラム名に起因するエラー
>Error when building FieldInfo from annotated attribute. Make sure you don't have any field name clashing with a type annotation.

原因は、**フィールド名が Python の組み込み名や型名とぶつかっている**ことです。`schemas.py` では次の2つが該当します。

```9:12:src/backend/schemas.py
    id: int
    date: date
    amount: int
    type: str
```

---

## 1. `date: date`（最有力）

```python
from datetime import date

date: date  # フィールド名も型も "date"
```

Pydantic v2 はクラス定義時に `date: date` を解釈しますが、

- 左の `date` … フィールド名
- 右の `date` … 型（`datetime.date`）

同じ名前のため、「フィールド名と型注釈が衝突している」と判断され、今回のエラーになります。

---

## 2. `type: str`

```python
type: str
```

`type` は Python の組み込み関数（`type(123)` の `type`）です。フィールド名に使うと、同様の衝突エラーになりやすいです。`TransactionCreate` の 21 行目も同じです。

---

## なぜ `models.py` では問題ないのか

SQLAlchemy は `type = Column(...)` のように **代入** で定義するため、型注釈の `type: str` とは構文が違います。Pydantic は `名前: 型` という型注釈を解析するので、この問題が起きます。

---

## 解決方法

### 方法A: `Annotated` + `Field` を使う（名前はそのまま）

```python
from typing import Annotated
from pydantic import Field

class TransactionRead(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    id: int
    date: Annotated[date, Field()]
    amount: int
    type: Annotated[str, Field()]
    description: str | None
    created_at: datetime
    updated_at: datetime
```

`Annotated` で「これは型注釈である」と明示し、衝突を避けます。

### 方法B: Python 側の属性名を変え、JSON 名は `alias` で合わせる

```python
class TransactionRead(BaseModel):
    model_config = ConfigDict(from_attributes=True, populate_by_name=True)

    transaction_date: date = Field(alias="date")
    transaction_type: str = Field(alias="type")
    # ...
```

ORM の `date` / `type` カラムとは `model_validate` 時に `from_attributes=True` でマッピングされます（属性名が違う場合は `Field(validation_alias="date")` などが必要になることもあります）。

---

## 補足: `TransactionCreate` の `class Config`

```python
class Config:
    from_attributes = True
```

これは Pydantic v1 の書き方です。v2 では `model_config = ConfigDict(from_attributes=True)` に統一するのがよいです。今回のエラーの直接原因ではありませんが、合わせて直すとよいです。

---

**まとめ:** エラーは DB や Docker ではなく、Pydantic のフィールド定義の問題です。特に `date: date` と `type: str` の名前衝突が原因です。`Annotated` を使うか、属性名を変えて `alias` で JSON/DB 名を合わせてください。
